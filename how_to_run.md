# 🚀 How to Run: Secure Feedback System

Follow these steps every time you start the project to ensure all components are connected correctly.

## 1. Start Core Services
The system depends on two main services. Ensure they are running before starting the app.

### 🍃 MongoDB
- **Purpose**: Primary database for users, feedback, and logs.
- **Run**: Open your MongoDB Shell or Compass, or ensure the service is running.
- **Port**: `27017`

### ⛓️ Blockchain (Ganache)
- **Purpose**: Immutable ledger for feedback integrity.
- **Run**: Use a **seed** to keep your accounts the same every time:
  ```bash
  npx ganache --seed 1234
  ```
- **Port**: `8545`
- **Check `.env`**: Ensure `ACCOUNT_ADDRESS` and `PRIVATE_KEY` match the first account generated by this seed.

---

## 2. Deploy Smart Contract (First Time or Reset)
If you have just started a new Ganache workspace or changed the contract:

```bash
python deploy.py
```
*This will update `contract_artifacts.json` with the new address.*

---

## 3. Run the Flask Application

```bash
python app.py
```

- **Access**: `http://127.0.0.1:5000`
- **Identity Key**: On the first run, `secret.key` is generated. **Do not delete it**, or you won't be able to decrypt existing identities.

---

## ✅ Persisting Your Data Across Sessions

You can safely close VS Code, shut down the computer, or reopen the project later –
nothing is lost *as long as the blockchain and database are preserved*.

1. **MongoDB** runs as a separate service; your feedbacks are stored there.  Leave
   the service installed and started (e.g. via `mongod` or Windows service).

2. **Ganache state** lives in its own process. When you reopen the project:
   * Start Ganache exactly the same way you did before.  Use a fixed seed or the
     GUI workspace so the accounts don’t change.
   * For long‑term work, run Ganache with a persistent database directory:

     ```bash
     npx ganache --db ./ganache-db --seed 1234 --port 8545
     ```

     This ensures contract storage survives restarts; you can shutdown and restart
     the CLI/GUI without losing feedbacks.

3. **Contract artifacts** (`contract_artifacts.json`) and your `.env` file should
   remain in the project folder.  Do **not** rerun `deploy.py` unless you intend
   to start from scratch.

4. After starting MongoDB and Ganache, run `python app.py` as usual.  The app will
   load the existing contract and display all previously recorded feedback.

5. You can verify the on‑chain ledger at any time with:

   ```bash
   python check_blockchain_state.py
   ```

Keeping these components running (or using Ganache’s database option) is all you
need to “pick up where you left off” when reopening the workspace later.
